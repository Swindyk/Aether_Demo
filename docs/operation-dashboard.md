# 后台操作手册

后台用于查看会话、运行详情、知识来源、Skill、Trace 和错误记录。普通玩家日常使用首页即可；后台主要用于演示、排障和复盘。

![后台](assets/dashboard.png)

## 入口

- 首页右上角设置按钮。
- 系统托盘菜单中的 `打开后台`。

## 页面结构

- 左侧：全部会话列表。
- 右侧顶部：当前会话摘要、模型、请求编号、耗时、账号和轮次。
- 右侧标签页：总览、视觉观察、知识、Skill、Trace、错误/后台。

## 常用操作

### 查看历史会话

1. 在左侧点击一条历史会话。
2. 右侧详情应同步切换到该会话。
3. 核对最终回答、视觉观察、知识来源、Skill 和 Trace 是否都跟随切换。

### 继续追问历史会话

1. 选中目标会话。
2. 在右侧追问输入框中输入问题。
3. 发送后应立即出现玩家消息和 `以太思考中......`。
4. 返回后，新回答会追加到同一会话。

### 查看知识来源

打开 `知识` 标签页，重点看：

- 本地知识命中。
- 联网搜索 query。
- 实际采用来源。
- 被过滤来源。
- 本轮检索策略。

攻略类问题通常应优先联网检索；只有高置信本地 QA 命中时才直接使用本地知识回答。

### 查看 Skill 与 Trace

- `Skill`：查看每个能力模块是否触发、为什么触发、输入输出摘要和耗时。
- `Trace`：查看完整执行时间线，用于定位卡顿、模型失败或检索失败。

### 删除历史

后台支持删除单条会话，也支持清空当前账号或全部历史。删除会同步清理本地 conversation 和关联运行记录；执行前确认不再需要这些演示样例。

## 排障建议

### 后台显示模型不可用

检查 `.env.local` 或 `.env`：

```text
AETHER_LLM_PROVIDER=Sub2Api
AETHER_LLM_BASE_URL=http://127.0.0.1:8080/v1
AETHER_LLM_WIRE=responses
AETHER_LLM_MODEL=gpt-5.5
AETHER_LLM_API_KEY=你的本地 Sub2API API Key
```

同时确认 Sub2API Docker 服务正在运行。

### 回答里没有来源

先看 `知识` 标签页。如果本轮不是攻略意图，可能只基于视觉观察回答；如果是攻略意图但没有来源，检查 Tavily key、网络状态和过滤来源。

### 历史会话点击后详情不变

这是需要优先修复的严重问题。正常行为是左侧会话选择后，右侧最终回答、视觉观察、知识、Skill 和 Trace 全部同步切换。

### 旧历史里出现乱码

旧版本写入过的坏数据可能仍在本地历史中。当前界面会尽量用占位文案兜底；如果不需要保留旧样例，可以在后台删除对应会话或清空历史。

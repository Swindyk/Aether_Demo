# AgentOps 后台操作手册

AgentOps 是以太的演示、排障和复盘入口，用于查看会话、视觉观察、知识命中、Skill、Trace、错误和运行环境状态。玩家日常使用只需要首页；后台主要服务于答辩、作品集展示和调试。

## 主要区域

- **会话列表**：展示全部历史会话，点击后右侧详情会切换到对应会话。
- **总览**：查看最终回复、最近消息、运行结果来源和关键指标。
- **视觉观察**：查看截图识别摘要、场景、OCR 文本、可见事实和置信度。
- **知识库**：查看本地知识卡、联网来源、采用来源和被过滤来源。
- **Skill**：查看本轮触发或跳过的能力模块，以及触发原因、输入摘要、输出摘要和耗时。
- **Trace**：查看从观察、检索、推理到回答的完整时间线。
- **错误/后台**：查看失败记录、边界规则、运行环境、知识库版本和数据目录。

## 标准排障闭环

### 1. 打开目标会话

1. 在左侧会话列表中选择目标会话。
2. 确认右侧详情已切换到该会话，而不是仍停留在最新会话。
3. 核对当前会话标题、账号 key、最近消息和运行时间。

### 2. 复查回答依据

1. 打开“视觉观察”，确认以太实际看到了哪些画面事实和 OCR 文本。
2. 打开“知识库”，确认是否命中了本地知识卡、是否使用联网来源、是否过滤了低质量来源。
3. 打开“Skill”，确认公开账号、知识检索、来源过滤等模块是触发、跳过还是失败。
4. 打开“Trace”，按耗时和顺序定位慢步骤或异常步骤。

### 3. 在后台继续追问

1. 选中目标会话后，在右侧追问输入框中输入问题。
2. 发送后，结果应追加到当前会话，不应创建新会话。
3. 如果追问失败，查看“错误/后台”和“Trace”确认失败阶段。

### 4. 管理历史记录

- 单条删除：删除指定会话及其相关运行快照。
- 按账号清空：清理某个 UID 或账号 key 下的历史会话，可选是否同时清理 memory。
- 全部清空：清理全局历史。演示前应提前确认，避免误删需要展示的样例。

## 录屏演示建议（1 - 2 分钟）

1. 从首页完成一次 `Alt+Q` 解读。
2. 打开 AgentOps，点开刚生成的会话。
3. 依次展示“视觉观察”“知识库”“Skill”“Trace”。
4. 在后台追问一句“再给我一个保守方案”。
5. 展示回答追加到同一会话，并说明后台用于校验模型建议的来源和过程。

## 注意点

- 后台展示的是排障信息，不应作为玩家主界面的默认体验。
- 如果旧运行记录被清理，仍可查看会话消息流，但部分 run 级 trace 可能不可用。
- 如果知识库没有命中，先确认问题是否属于游戏场景，再检查 Tavily key、联网状态和来源过滤结果。
- 如果界面显示的模型名与配置不同，以模型服务实际返回的 `model` 字段为准。

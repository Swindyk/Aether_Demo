# 以太录屏与素材清单

## 体验审查结论

新版以太改为按需热键问答器，主打稳定、少打扰和容易录清楚：

- 首次启动打开正常主界面，不再依赖透明宠物窗口作为入口。
- `Alt+Q` 触发一次截图与 Agent 分析；默认读取鼠标所在屏幕。
- 分析完成后只显示短答案卡，不抢焦点，约 8 秒后自动收起。
- `Alt+Shift+Q` 打开完整回答与来源。
- 系统托盘提供“解读当前画面、查看最近回答、打开以太、退出以太”。
- AgentOps 继续展示真实请求编号、skill、RAG、memory、rules 与 trace。

## 70 秒录屏结构

建议主片控制在 70 秒以内，遵循“卡点 → 按键 → 短答案 → 证据链”。

| 时间 | 画面 | 重点 |
| --- | --- | --- |
| 0-8 秒 | 玩家卡点 | 字幕：卡点了 不想切屏查攻略 |
| 8-18 秒 | 按 `Alt+Q` | 短答案卡出现 不抢焦点 |
| 18-35 秒 | 装备或配队建议 | 第一行结论清楚 下面最多两步 |
| 35-48 秒 | 切换新手 进阶 剧情偏好 | 同一画面回答明显不同 |
| 48-60 秒 | 连接公开 UID | 建议变成基于角色池 |
| 60-70 秒 | AgentOps 证据链 | 截图理解 账号 知识 skill rules 请求编号 |

主片先讲玩家价值，AgentOps 只占最后 8 到 10 秒。

## 当前缺少的素材

优先级从高到低：

1. 真实游戏录像：1920×1080、60fps、无边框窗口，分别录装备判断、阵容选择、剧情对话、地图探索，每段 10 到 15 秒。
2. 问题前后对比片段：一段“切出游戏搜索攻略”，一段“按 Alt+Q 直接得到答案”。
3. 视频封面与品牌主视觉：16:9 横版、9:16 竖版、1:1 社交头像。
4. 固定口号：建议使用“少切一次屏，多玩一会儿”。
5. 干净产品截图：主界面、短答案卡、完整回答、AgentOps 各一张。
6. 3 到 5 秒架构动效：截图进入 Agent，经 memory、RAG、skill、rules 后返回建议。

不要使用虚假用户数量、虚假识别准确率或伪造评价。真实请求和真实画面本身已经足够有说服力。

## 仍需继续优化的产品风险

- 真实推理等待偏长：建议录制前使用真实调用确认关键场景，正式作品可明确标注“真实历史结果回放”。
- 受保护画面仍有限制：独占全屏或反作弊保护画面可能无法截图，录制时使用无边框窗口模式。
- 语音链路尚未实现：当前版本只使用提示音和短答案卡。
- Overwolf 未接入：后续若要真正游戏内 Overlay，需要单独走 Overwolf 客户端与开发者审批。

## 参考的成熟体验

- [Xbox Gaming Copilot](https://www.xbox.com/en-US/gaming-copilot)：强调更快回到游戏、随时求助与语音快捷唤醒。
- [Overwolf：In-game overlays](https://dev.overwolf.com/ow-native/guides/product-guidelines/app-screen-behavior/in-game-overlays/)：overlay 应增强体验而不打断游戏。
- [Mobalytics App](https://mobalytics.gg/lol/glp/app-download)：围绕赛前、赛中、赛后相关建议来宣发。
- [NVIDIA G-Assist](https://www.nvidia.com/en-us/software/nvidia-app/g-assist/)：用真实游戏画面叠加清晰问题和结果，快速证明价值。
